## 🔄 데이터 흐름

### 1️⃣ 웹 요청

```
사용자 
  ↓ HTTP/HTTPS 요청 (http://localhost:8000 또는 https://domain.com)
Cloudflare Tunnel (있는 경우)
  ↓ HTTP 라우팅
Podman Container
  ↓ 포트 8000 수신
Gunicorn Worker
  ↓ WSGI 인터페이스
Django View
  ↓ 데이터 조회/저장
데이터베이스 (SQLite)
  ↓ 응답
Django Template 렌더링
  ↓ HTML + 정적 파일
사용자 브라우저
```

### 2️⃣ 알고리즘 실행

```
사용자: 시뮬레이션 생성 요청
  ↓
Django View (api/views.py)
  ↓
Celery Task 생성
  ↓
Celery Worker (컨테이너 내부)
  ↓
최적화 알고리즘 실행 (Gurobi 등)
  ↓
결과 생성 (outputs/ 폴더에 저장)
  ↓
상태 업데이트
  ↓
사용자에게 알림
```

### 3️⃣ 데이터 보존

```
컨테이너 삭제
  ↓
Podman Volume 유지
  ↓
호스트 PC 폴더에 저장된 데이터 유지
  ↓
새 컨테이너 생성
  ↓
이전 데이터 접근 가능
```

```
  ↓
```
```
Podman Container
```

## 📈 확장성 고려사항

### 현재 구성 (Podman)

```
✅ 장점:
- 로컬 전체 리소스 사용 가능
- 최적화 연산에 충분한 메모리
- 높은 성능

❌ 제약:
- 하나의 로컬 PC에만 배포
- 낮은 가용성 (PC 꺼지면 서비스 중단)
```

### 향후 확장 (클라우드 전환 시)

```
고려 사항:
1. 메모리 충분한 서버 선택 (최소 4GB)
2. PostgreSQL로 DB 이전
3. Redis로 Celery 큐 구성
4. 로드 밸런싱 (여러 인스턴스)
5. 자동 백업 필요
```

## 🚀 배포 흐름도

```
코드 변경
  ↓
.env.docker 수정 (필요시)
  ↓
Dockerfile로 이미지 빌드
  ↓
이미지에 requirements.txt 포함
  ↓
run-docker 스크립트 실행
  ↓
기존 컨테이너 정지/삭제
  ↓
새 컨테이너 생성
  ↓
docker-entrypoint.sh 실행
  ├─ DB 마이그레이션
  ├─ 정적 파일 수집
  └─ Gunicorn 시작
  ↓
웹 서비스 시작 (포트 8000)
  ↓
로그인 및 테스트
```
